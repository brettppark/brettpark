# AREPO SN Ia Reference Matrix

<span style="color:red"><b>Status:</b> this is the reference map for a research-code implementation. It is not evidence that the current public AREPO tree already contains publishable SN Ia explosion physics.</span>

## Decision Summary

The best first physical target for our AREPO branch is DD/D6/sub-Chandrasekhar detonation physics, not a Chandrasekhar-mass SD delayed detonation. The reason is technical and scientific: DD/D6 needs merger dynamics, hotspot/edge-lit detonation criteria, burning/source-term coupling, and ejecta export. SD Chandrasekhar DDT additionally requires an unresolved turbulent deflagration flame model plus a calibrated DDT prescription, which is a larger and more ambiguous implementation.

The code branch now keeps both pathways visible: DD/D6/sub-Chandra and SD-DDT compile as separate diagnostics-only AREPO configurations. The physical implementation order is still DD/D6 first, SD-DDT second. AREPO should be developed as the hydro/explosion/ejecta-export stage. It does not remove the radiative-transfer stage. `M_B`, `x1`, and `c` require ARTIS/TARDIS/SEDONA-like RT plus SALT fitting.

## Implementation Choice

1. First publishable hydro target: DD/D6/sub-Chandra double detonation in AREPO.
2. First code target: build stable source-term, branch diagnostics, and export infrastructure before attempting physical detonation.
3. First validation target: reproduce scalar conservation, one-zone burn energetics, planar detonation speed/yields, and then a published sub-Chandra model.
4. SD target: keep MESA for long accretion/cooling; use AREPO only for dynamical ignition/explosion/ejecta-companion work after the WD state is defined. The current SD-DDT module boundary is intentionally diagnostics-only until the flame/turbulence/DDT literature choices are locked.
