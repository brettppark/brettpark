# AREPO Stellar Physics Equation Set For SN Ia WD Binaries

This note separates the physics that belongs in AREPO from the secular stellar
evolution physics that belongs in MESA/COMPAS.

## What AREPO Already Solves

The public AREPO branch solves moving-mesh finite-volume hydrodynamics with
self-gravity. In conservative form, the active gas equations are:

```text
d rho/dt + div(rho v) = 0
d(rho v)/dt + div(rho v v + P I) = -rho grad Phi
d(rho E)/dt + div[(rho E + P) v] = -rho v . grad Phi + source terms
laplacian Phi = 4 pi G rho
```

The mesh is generated by moving Voronoi cells, so the calculation is
hydrodynamic, not an N-body calculation. Current SN Ia branch additions only
advect passive composition scalars and write diagnostics.

## WD Physics That Must Be Added Before Publication Runs

1. **Degenerate EOS.** Replace the ideal-gas closure with a Helmholtz-style EOS:
   ions + radiation + arbitrarily degenerate/relativistic electrons/positrons.
   Reference: Timmes & Swesty 2000, doi:10.1086/313304.
2. **Hydrostatic WD construction and relaxation.** MESA/WDEC profiles must be
   mapped so that `grad P ~= -rho grad Phi`, then damped in isolation before
   being placed into a binary.
3. **Mesh refinement policy.** Refine by target cell mass and protect WD
   surfaces, Roche streams, hot spots, and detonation fronts from derefinement.
4. **GW/inspiral driver.** Long detached inspiral should not be integrated
   literally. Use a documented angular-momentum loss prescription until contact,
   then disable it for the dynamical merger unless a sensitivity test requires
   otherwise. Burmester et al. 2023 discuss accelerated inspiral in AREPO
   WD merger simulations, doi:10.1093/mnras/stad1394.
5. **RLOF and binary diagnostics.** The mass transfer itself is hydrodynamic,
   but the code must measure Roche geometry, L1 flow, component masses,
   separation, angular momentum, and bound/unbound material.
6. **Nuclear source terms.** Add energy-coupled alpha/reduced networks for
   hydro, and tracer export for larger post-processing networks.
7. **Detonation criteria.** DD/D6 needs cited helium/core detonation triggers;
   SD needs flame, turbulence, and DDT criteria.

## What Should Not Be Moved Into AREPO

Do not evolve the full ZAMS-to-WD lifetime in 3D hydro. Main-sequence evolution,
common-envelope bookkeeping, WD cooling over Myr-Gyr, and population weights
belong to COMPAS/MESA/Paper I. AREPO starts from a compact WD binary state:
detached late inspiral, Roche contact, mass transfer, merger, and explosion.

## Current Blocking Point

Increasing cell count improves spatial sampling, but it does not make a stable
WD if the EOS and hydrostatic relaxation are wrong. The immediate implementation
target is therefore:

```text
Helmholtz EOS wrapper -> isolated WD relaxation -> binary relaxation ->
GW/contact driver -> merger diagnostics -> burning/detonation
```
